title:《电子商务数据分析》笔记2

第二章 电商数据分析岗位必备的技能

第一节 初识电商行业的大数据

本书核心价值：探讨和学习如何借助大数据服务于电商行业经营管理。

1.1 借助大数据分析优化市场定位

对外：

- 拓宽电商行业调研数据的广度和深度；
- 从大数据中了解电商行业**市场构成、细分市场特征、消费者需求**和**竞争者状况**等众多因素；

对内：

- 项目评估和可行性分析；

传统分析资料的收集主要来自于统计年鉴、行业管理部门数据、相关行业报告、行业专家意见及属地市场调查。
缺点：样本量不足、时间滞后、准确度低

1.2 借助大数据优化市场营销

对外：

-  通过获取数据并加以统计分析充分了解市场信息，掌握竞争者的商情和动态，知晓产品在竞争群中所处的市场地位。

对内：

- 通过积累和挖掘电商行业消费者数据，分析顾客消费行为和价值趋向

对于电商行业，消费行为方面的数据，如：

- 消费者购买产品的花费
- 选择的产品渠道
- 偏好产品的类型
- 产品使用周期
- 购买产品的目的
- 消费者家庭背景
- 工作和生活环境
- 个人消费观和价值观

1.3 大数据助力电商企业的收益管理

**收益管理** ：起源于20世纪80年代的一种谋求收入最大化的新经营管理技术，意在把合适的产品或服务，在合适的时间，以合适的价格，通过合适的销售却道出售给合适的顾客，最终实现企业收益最大化的目标。要达到收益管理的目标，**需求预测、细分市场和敏感度分析**是工作中的三个重要环节。
**需求预测**：通过对建构的大数据进行统计和分析，采用课学的预测方法，通过建立数学模型，使企业管理者掌握和了解行业潜在的市场需求，未来一段时间每个细分市场的产品销售量和产品价格走势。
**细分市场**：为企业预测销售量和示星差别定价提供了条件，其科学性体现在通过电商行业市场需求预测来指定和更新价格，使各个细分市场的收益最大化。
**敏感度分析**： 通过需求价格弹性分析技术，对不同细分市场的价格进行优化，最大限度低挖掘市场的潜在收入。

1.4 大数据协助创新用户新的需求

第三节 电商数据分析的方法和流程

**核心词汇**：
    1. 独立用户访问量：就是常说的UV，即有多少台计算机在24小时内访问网站（UV和IP并不相同）
    2. 积极访问者比率：如果你得网站针对正确的目标受众并且网站使用方便，你可以看到这个指标应该是不断上升的。
    3. 忠实访问者比率：每个长时间访问者的平均访问页数，这是一个重要指标，它结合了页数和时间。
    4. 客户转化率：转化率指在一个统计周期内，完成转化行为的次数占推广信息总点击次数的比率。
    5. 客单价：每一个顾客平均购买商品的金额，即平均交易金额。
    6. 客户满意度：客户期望值与客户体验的匹配程度。
    7. 用户回访率：衡量网站内容对访问者的吸引程度和网站的实用性。
    8. 投资回报率：用来衡量营销费用的投资回报，把钱分配给最高回报率的营销方式。

3.1 收集数据 

1. 网站后台的数据
    - 网站用户数据（注册时间、用户性别、所属地域、来访次数、停留时间等）
    - 订单数据（下单时间、订单数量、商品品类、订单金额、订单频次等）
    - 反馈数据（客户评价，退货换货、客户投诉等）
2. 搜索引擎的数据
    - 网站在各个搜索引擎的收录量
    - 网站在搜索引擎的更新频率
    - 关键词在搜索引擎的竞价排名情况
    - 网站取得的搜索引擎新人的权重
3. 统计工具的数据
    - 访客来自哪些地域
    - 访客来自哪些网站
    - 访客来自哪些搜索词
    - 访客浏览了哪些页面
4. 调查问卷收集的数据

3.2 量化分析

1. 趋势分析：
    - 定比：以某一时期为基数，其他各期均与该期的基数进行比较。
    - 环比：分别以上一期为技术，下一时期与上一时期的技术进行比较。
2. 对比分析：
    - 把两个相互联系的指标数据进行比较，从数量上展示和说明对象规模的大小，水平高低、速度快慢，以及各种关系手否协调。
3. 关联分析：
    - 如果两个或多个事物之间存在一定的关联，那么其中一个事物能够通过其他事物进行预测；它的目的是为了挖掘隐藏在数据之间的相互关系。
4. 因果分析：
    - 研究对象的先行情况中，把作为它的原因的现象和其他非原因的现象区分开来。

3.3 提出方案

1. 评估描述：对评估情况进行客观描述，用数据支持自己的观点；
2. 编制统计图表： 运用柱状和条形图对基本情况进行更清晰的描述；运用散点图和折线图变现数据间的因果关系；
3. 提出观点：根据现实情况的数据分析，提出自己的观点，预判网站的发展趋势，给出具体的建议性的改进措施。
4. 演示文档：基于以上三点进行归纳总结，列出条目，制作一份详细的演示文档，能够演示和讲解给部门领导。
